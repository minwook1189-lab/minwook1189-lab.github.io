### 시연 1: 고객용 대시보드 (5분) ⭐ 하이라이트

**스토리라인:**
> "고객사에서 본인들 물량 진행 상황을 실시간으로 보고 싶다고 합니다.
> 개발팀에 요청하면 몇 달 걸릴 텐데, AI로 지금 바로 만들어보겠습니다."

**시연 순서:**

1️⃣ **빈 폴더에서 시작** (Cursor에서)
   - 새 폴더 `demo_live/` 생성
   - Cursor로 열기

2️⃣ **AI에게 대시보드 생성 요청** (라이브로)
   - 아래 프롬프트 복사해서 사용

3️⃣ **결과 확인**
   - 브라우저에서 index.html 열기
   - 차트, 테이블 확인

4️⃣ **즉석 수정 요청** (라이브로)
   - 아래 수정 시나리오 중 1개 실행

**📝 초기 프롬프트 (복사해서 사용)**

```
반도체 파운드리 고객 포털 대시보드 1페이지 만들어줘.

- 헤더: SemiFab 로고 + Samsung Electronics 로그인 표시
- 4개 지표 카드 (진행 중 Lot 127, 수율 94.7%, 출하예정 45, Cycle Time 18.5일)
- Chart.js 월별 생산량 라인 차트 (생산량, 출하량 2개 라인)
- Lot 진행 현황 테이블 3행 (진행률 바, 상태 뱃지 포함)
- 파란색 테마, 깔끔한 카드 디자인
```

**📝 즉석 수정 시나리오 (생성 후 1개 선택)**

| 요청 | 예시 프롬프트 |
|------|---------------|
| 회사명 변경 | *"회사명을 SK키파운드리로 바꿔줘"* |
| 사용자 변경 | *"로그인 사용자를 '김민욱'으로 바꿔줘"* |
| 색상 변경 | *"메인 색상을 빨간색 계열로 바꿔줘"* |

**핵심 포인트:**
- **처음부터 생성**: 빈 폴더에서 대시보드가 만들어지는 과정 시연
- 코딩 없이 자연어 요청만으로 즉각 생성/수정 가능
- 기획자/현업이 직접 프로토타입 생성 가능

**참고 파일:**
- [05_demo_site_spec.md](05_demo_site_spec.md) - 웹사이트 스펙 문서 (압축 버전)

### 시연 2: 데이터 시각화 (5분)

**스토리라인:**
> "현업에서 엑셀로 관리하는 데이터가 있죠? 이걸 웹 차트로 시각화해보겠습니다."

**시연 순서:**

1️⃣ **Excel 파일 준비**
   - `sample_production_data.csv` 파일을 demo_live/ 폴더에 복사
   - (또는 시연 1에서 만든 웹사이트 폴더에 복사)

2️⃣ **AI에게 시각화 요청**
   - *"이 데이터로 막대 차트 만들어줘"*

3️⃣ **결과 확인**
   - 브라우저에서 생성된 차트 확인

4️⃣ **추가 요청 (선택)**
   - 다른 차트 유형으로 변경
   - 스타일 수정

**📝 시각화 프롬프트 시나리오**

| 요청 | 예시 프롬프트 | 비고 |
|------|---------------|------|
| 기본 막대 차트 | *"이 데이터로 막대 차트 만들어줘"* | ⭐ 메인 |
| 라인 차트 추가 | *"수율 데이터를 라인 차트로 추가해줘"* | 추가 시연 |
| 차트 스타일 | *"차트 색상을 파란색 계열로 바꿔줘"* | 쉬움 |
| 복합 차트 | *"생산량은 막대, 수율은 라인으로 같이 보여줘"* | 고급 |

**📊 샘플 데이터 구조** (sample_production_data.csv)

| 월 | 생산량 | 출하량 | 목표 | 수율 |
|----|--------|--------|------|------|
| 1월 | 4,500 | 4,200 | 4,000 | 94.2% |
| 2월 | 4,800 | 4,500 | 4,200 | 93.8% |
| ... | ... | ... | ... | ... |
| 12월 | 7,500 | 7,200 | 7,000 | 97.1% |

**핵심 포인트:**
- 엑셀 데이터를 웹 차트로 즉시 변환
- 비개발자도 데이터 시각화 가능
- 다양한 차트 유형 요청 가능
- 🔗 **시연 3 연계 예고**: *"이 차트를 매일 자동으로 메일로 받아볼 수 있습니다. 바로 이어서 보여드리겠습니다."*

**참고 파일:**
- [sample_production_data.csv](sample_production_data.csv) - 샘플 생산 데이터

### 시연 3: 메일 자동화 (5분) ← 시연 1,2와 연계

**스토리라인:**
> "방금 만든 생산 현황 그래프, 매일 아침 자동으로 메일로 받아보면 어떨까요?
> 시연 2에서 만든 차트를 매일 특정 시간에 자동 발송하도록 설정해보겠습니다."

**시연 순서:**

1️⃣ **연계 포인트 설명**
   - *"방금 시연 2에서 sample_production_data.csv로 차트를 만들었죠?"*
   - *"이 차트를 이미지로 저장해서 매일 아침 메일로 보내면 어떨까요?"*

2️⃣ **AI에게 자동화 요청**
   - *"시연 2에서 만든 생산 현황 차트를 이미지로 저장하고, 매일 아침 8시에 특정 메일 주소로 자동 발송하는 스크립트 만들어줘"*

3️⃣ **스크립트 생성 과정 확인**
   - 차트 이미지 생성 코드
   - Outlook 메일 발송 코드
   - 스케줄링 설정

4️⃣ **테스트 실행**
   - 스크립트 실행해서 `email_preview.html` 열어보기
   - 실제 메일 발송 테스트 (선택)

5️⃣ **조건 추가 (선택)**
   - *"수율이 93% 미만일 때만 긴급 알림 보내게 해줘"*
   - *"차트에 목표 대비 달성률도 표시해줘"*

**📝 메일 자동화 프롬프트 시나리오**

| 요청 | 예시 프롬프트 | 비고 |
|------|---------------|------|
| 기본 자동화 | *"시연 2에서 만든 생산 차트를 이미지로 저장하고 abc@company.com으로 매일 아침 8시에 보내는 스크립트 만들어줘"* | ⭐ 메인 |
| 조건부 발송 | *"수율이 93% 미만이면 '긴급' 표시해서 보내줘"* | 추가 시연 |
| 다중 수신자 | *"팀장들 메일 주소 여러 개로 보내게 해줘"* | 확장 |
| 주간 리포트 | *"주간 단위로 요약해서 월요일마다 보내줘"* | 고급 |

**핵심 포인트:**
- 🔗 **시연 1→2→3 연계**: 대시보드 → 데이터 시각화 → 자동 리포팅의 흐름
- 반복 업무 자동화로 시간 절약
- 비개발자도 AI 도움으로 자동화 구축 가능
- 조건부 알림으로 중요 이슈 즉시 파악

**참고 자료:**
- [03_daily_report_email.py](03_daily_report_email.py)
- [04_schedule_email_guide.md](04_schedule_email_guide.md)
- [sample_production_data.csv](sample_production_data.csv) - 시연 2와 동일 데이터 사용

## 💡 시연 팁

### Do's ✅
- **실시간으로 보여주기**: 미리 만든 것 아닌 걸 강조
- **실패도 보여주기**: 100% 완벽하지 않음, 사람의 검토 필요
- **즉석 요청 받기**: "뭐 바꿔볼까요?" 관객 참여 유도
- **속도 강조**: "원래라면 며칠 걸릴 작업"

### Don'ts ❌
- 너무 기술적인 설명 피하기
- "AI가 다 해줍니다" 과장 금지
- 민감한 회사 데이터 사용 금지

## 🎤 예상 질문 & 답변

**Q: 보안은 괜찮은가?**
> 회사 데이터를 외부 AI에 직접 넣지 않고, 구조/형식만 참고합니다.
> 실제 데이터는 사내 시스템에서 처리하고, AI는 코드 작성만 도와줍니다.

**Q: 개발자가 필요 없어지나?**
> 프로토타입/POC는 빨라지지만, 실제 운영 시스템은 여전히 개발자 필요.
> 오히려 개발자의 생산성이 크게 향상됩니다.

**Q: 비용은?**
> Claude Pro 월 $20, Cursor Pro 월 $20
> 기업용 라이선스도 있음

**Q: 어디서 배우나?**
> 직접 써보는 게 최고의 학습. 오늘 만든 커리큘럼 참고.
