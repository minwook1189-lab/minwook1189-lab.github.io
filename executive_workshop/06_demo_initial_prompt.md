# 시연 1: 초기 프롬프트 가이드

## 시연 전 준비

1. **새 폴더 생성**: `demo_live/` (빈 폴더)
2. **Cursor에서 폴더 열기**
3. **05_demo_site_spec.md 파일을 폴더에 복사** (AI가 참조할 수 있도록)

---

## 권장 초기 프롬프트

### 옵션 A: 스펙 파일 참조 (가장 안정적)

```
05_demo_site_spec.md 파일을 읽고 이 스펙대로 반도체 파운드리 고객 포털 웹사이트를 만들어줘.
```

**장점:** 스펙 파일에 모든 내용이 있어서 결과물이 일관됨

---

### 옵션 B: 자연어로 직접 요청 (더 인상적)

```
반도체 파운드리 회사의 고객 포털 웹사이트를 만들어줘.

고객사(예: 삼성전자)가 로그인해서 자기 물량의 생산 현황을 실시간으로 볼 수 있는 대시보드야.

필요한 페이지:
1. 홈 - 대시보드 (핵심 지표 카드, 월별 생산량 차트, 진행 중인 Lot 테이블)
2. 생산현황 - Lot 목록 전체
3. 품질리포트 - 수율, Defect Density 등
4. 문의하기 - 담당자 연락처, 문의 폼

디자인은 깔끔하고 전문적인 느낌으로, 파란색 계열 메인 컬러.
차트는 Chart.js 사용해줘.
```

**장점:** 자연어만으로 만들어지는 모습이 인상적

---

### 옵션 C: 짧고 간단하게

```
반도체 파운드리 고객이 자기 물량 진행상황을 보는 포털 대시보드 웹사이트 만들어줘.
홈, 생산현황, 품질리포트, 문의하기 4개 페이지로.
Chart.js로 차트 넣고, 깔끔한 UI로 해줘.
```

**장점:** 빠름, 하지만 결과물 품질이 다를 수 있음

---

## 시연 직후 확인 포인트

AI가 생성을 완료하면:

1. **브라우저에서 index.html 열기** (Live Server 또는 직접 열기)
2. **페이지 전환 확인** - 네비게이션 클릭
3. **차트 확인** - Chart.js 차트가 나오는지
4. **반응형 확인** - 브라우저 창 크기 조절 (선택)

---

## 결과물이 다를 경우

AI가 생성한 결과물이 예상과 다를 수 있습니다. 이런 경우:

### 긍정적으로 활용
> "보시면 AI가 나름대로 해석해서 만들었는데요,
> 제가 원하는 대로 수정을 요청하면 됩니다."

### 즉시 수정 요청
- "로고를 왼쪽 상단에 넣어줘"
- "차트 색상을 파란색으로 바꿔줘"
- "Lot 테이블에 진행률 바를 추가해줘"

### 최악의 경우 백업
- `demo_site_before/` 폴더의 완성본 사용

## 스펙 파일 사용 팁

스펙 파일(05_demo_site_spec.md)을 같은 폴더에 넣어두면:

1. AI가 @05_demo_site_spec.md로 참조 가능
2. 더 정확한 결과물 생성
3. 시연 중 "스펙 문서를 미리 정리해뒀습니다"라고 설명 가능
